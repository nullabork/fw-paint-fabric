# FW Paint v1.2.1

FW Paint now runs on **NeoForge** as well as Fabric. Two jars below — grab the one matching your
loader. No gameplay changes since 1.2.0.

Targets: Minecraft **26.2** · Java **25** · Fabric (Loader 0.19.3+ with Fabric API for 26.2) **or**
NeoForge for 26.2.

## NeoForge build

- New `fw-paint-neoforge` jar: the full mod — all three paint tools, every placement and marker
  mode, previews, HUD, and the in-game Help — identical to the Fabric version. It has **no extra
  dependencies** (Fabric API is only needed on the Fabric side).
- Install: drop the jar for your loader into a 26.2 instance's `mods/` folder. Client-side only,
  as always — nothing to install on the server, and placement stays multiplayer-safe on both
  loaders.

## Under the hood

- The codebase is now multiloader: all game code lives in a loader-free `common` module (26.x is
  unobfuscated, so the identical classes run on both loaders) with thin Fabric/NeoForge
  entrypoint shells. Every release from here on ships both jars automatically.
- Config and markers live in the same `config/gradient.json` / `config/gradient_markers.json` on
  both loaders, so switching loader keeps your settings and saved markers.
