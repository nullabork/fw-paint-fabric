# FW Paint v1.1.1

Gradient quality release: the Deviation slider becomes **Variation** (band variation that never
costs you steps), placement is finally deterministic and even when you want it to be, markers count
as the gradient's first and last blocks, and a new **Step length** slider wobbles band lengths
without breaking the blend.

Targets (unchanged): Minecraft **26.2** · Fabric Loader **0.19.3+** · Fabric API for 26.2 · Java **25**.

## Variation (was Deviation) — gradient & noise tools

- The gradient now always builds the fullest ramp available (endpoint range + max steps + a fixed
  sanity filter that keeps wildly off-colour blocks out). The slider never adds or removes steps —
  at 0% you get the whole gradient, not an empty one.
- Instead, **Variation** widens each step's band: similar blocks (including ones the Max-steps cap
  folded out) can randomly stand in for the step at placement. Nothing close enough → the step
  stays clean; a far-off colour is never picked.
- Step rows in the block list show their swappable alternates as small icons — drag the slider and
  watch bands gain/lose members live.
- The noise tool gets the same treatment: its Deviation (an old ±1-step shift that could jump to a
  visibly different block) is now Variation with per-step band groups, shown in its list too.

## Even, predictable placement

- Removed a hidden per-placement "tie" randomiser that shuffled similar blocks regardless of any
  slider — at Variation 0 / Chaos 0 a fill is now fully deterministic, even bands.
- Fixed step rounding: bands are equal-width (floor-based) instead of rounded, so a monotone curve
  (e.g. Ease Out) always produces monotone band sizes — no more a middle step getting fewer blocks
  than an earlier one, and no under-represented first/last bands (this also slightly evens noise
  fills).
- **Markers are the gradient's first and last blocks**: the fill spans strictly between them, so
  the first placed block no longer duplicates the start block against the marker. Markers 4 apart
  with 5 steps = each block exactly once, markers included.
- The block list now shows the gradient's actual distinct steps (one row each, [S] → [E]) instead
  of a curve-simulated sequence that could skip or duplicate entries and mislead about what would
  be placed.

## New: Step length slider (gradient tool)

- A per-step chance (0–100%) that a step runs longer or shorter — implemented as boundary shifts,
  so a lengthened step automatically shortens its neighbour and the gradient still starts and ends
  on time. 100% wobbles every boundary. The wobble is rolled once per marker line (coherent while
  you hold to fill) and re-rolls on a new line.

## Small stuff

- Yellow hint under the noise preview grid ("Click + drag the preview to pan").
- Tagged releases now take their body from `docs/release-notes/<tag>.md` automatically.
