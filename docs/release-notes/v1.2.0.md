# FW Paint v1.2.0

The placement rework: what you paint and where you place it are now two independent choices —
every paint type works through every placement mode, with or without markers. Plus two new ways
to lay markers, long-range marking, and free-hand gradients that remember where they were.

Targets (unchanged): Minecraft **26.2** · Fabric Loader **0.19.3+** · Fabric API for 26.2 · Java **25**.

## Paint type × placement mode

- **V** (or the new **Use** button on each tool tab) switches what you paint: Solid / Gradient /
  Noise. **G** cycles where blocks go: **Marker → Marker corners → Marker draw → Single → Face →
  3D Fill → Disabled** — one global mode, no more per-tool cycles.
- Markers are no longer a prerequisite anywhere: they only constrain (columns stop at end
  markers even in air, 3D fills respect marker space) and give gradients their endpoints.
- **Single** — one column out of the clicked face. **Face** — every connected, reachable block
  on the clicked face's plane extrudes together (top of a wall = taller, side = thicker); with
  a start marker behind the click it's the connected marker group instead. **3D Fill** — the
  blob fill, now for all three paint types.
- A **green face tint + arrow** previews exactly which faces the next click grows from and in
  which direction — all of them in Face mode, every open side in 3D. While you hold, the
  preview locks to the operation and rides the advancing fronts.
- Holding to repeat now waits a beat (~200 ms) after the first layer, so a tap reliably places
  exactly one layer; keep holding and it runs at full speed.
- New Settings: **Fill voids first** (in-marker Face fills level the lowest reachable columns
  before stacking — on by default) and one **Placement** button replacing the three per-tool
  mode buttons.

## Gradients anywhere

- Painting outside a marker line now works: the gradient comes from the picker list, one block
  per step, and a column simply stops when its gradient is done.
- **Gradient memory**: placed blocks remember their step, so re-clicking a half-finished
  gradient continues it. The memory clears when the picker gradient changes or after an idle
  timeout (new Settings slider, 10 s – 5 min, default 1 min).
- **3D gradients**: spheres blending from the centre outward. Each fill remembers its centre —
  click a block that belongs to one and it keeps growing the same sphere.
- Endpoints resolve per side: a marker sitting in air falls back to the picker block for that
  side only. The helper text always tells the truth — "Selected from markers", "Selected from
  picker", or a split start/end pair.

## New marker modes

- All marker modes now reach out to the **Marker dist** setting — markers aren't blocks, so
  you can mark far beyond block-placing range.
- **Marker corners**: left-click one corner (its face sets the column direction), a thin box
  follows your aim, right-click the opposite corner — start markers fill the first plane, end
  markers the opposite one, a colinear pair per column. With Auto end marker on, each column
  hugs the terrain instead, and columns with nothing paintable inside the box are skipped.
- **Marker draw**: a freehand pencil — hold and sweep to spatter start markers (right-click
  for ends); auto ends scan along the face the stroke began on. Strokes starting on a marked
  block erase.
- **Clear connected** (new keybind, default L-Ctrl): hold it while clicking a marked block to
  remove its whole connected plane at once. Clicking marked blocks toggles them off in every
  marker mode.

## Small stuff

- The K screen opens on the tab of whatever you're painting.
- The Help tab covers all of the above and always shows your **current** key bindings, even
  after rebinds.
- Upgrade note: the old per-tool mode settings are replaced by one placement mode, which
  starts as Disabled — press G once after updating.
