# FW Paint v1.1.0

The "one tool, three paints" release: the gradient and noise tools merge into a single configurable
paint tool with a paint-type keybind, a brand-new **Solid** tool (extrude + 3D fill) joins them, the
helper text becomes movable, and the colour-mode maths gets two correctness fixes.

Targets (unchanged): Minecraft **26.2** · Fabric Loader **0.19.3+** · Fabric API for 26.2 · Java **25**.

## One paint tool

- The separate gradient/noise tool items are replaced by a single **Paint tool** (Settings tab).
  Existing configs migrate automatically — your old gradient tool item (or noise item, if that was
  the only one set) becomes the paint tool.
- New keybind **V** switches the active paint type — Solid → Gradient → Noise — while holding the
  tool. The choice is persisted. **G** still cycles the active type's mode, and each type remembers
  its own mode.

## New: Solid tool

- **Extrude mode** — right-click a face to build a straight row/column out of it; hold to keep
  going until blocked or out of reach. Click a face of connected, coplanar start markers to extrude
  the whole wall at once; re-clicking a half-built wall continues it (each column resumes from its
  own first air block). Scattered end markers stop only the column that hits them.
- **3D Fill mode** — a connected fill that grows outward from the clicked face, layer by layer
  while held (tap for one small blob). It spreads through air only, so walls contain it. Start
  markers cast a constraint space from all six faces: a fill started inside can't leave, one
  started outside can't enter; end markers stop it physically.
- **Match modes** — what gets placed: the ✓ **Selected block**, the **Exact block** you click, or
  the **Closest colour / Closest brightness** block from your source. Excluded blocks are never
  placed — exact-matching one flashes a warning above the hotbar.
- Solid tab with its own picker: ✓ single-selects the block to place (auto-disarms), ✗ toggles
  exclusions (stays armed). The ✓ row shows struck-through while a non-Selected match mode is
  active. The Solid tab is now first, and the default when the screen opens.

## Helper text (HUD)

- Now two lines — active paint type on top, mode below — shown only while holding the tool.
- New **Move helper text…** screen (Settings tab): arrow buttons with a cycling 50/20/10/1 px step
  and a live preview, in case the text covers something else on your screen.

## Gradient tool

- New **From: Markers / Block list** toggle (with a yellow description line): endpoints from the
  real blocks at your markers (old behaviour, default) or exactly the [S]/[E] preview list from the
  screen, so placement matches the preview regardless of what the markers are made of.

## Colour-mode fixes

- The **Deviation** slider now actually does something in Brightness / Top % Dark / Top % Light
  modes: deviation is measured as colour distance to the start→end line for every mode, so an
  off-colour block whose brightness happens to fit is dropped when the slider is tight. (The two
  Diff modes are inherently one-dimensional and are unaffected; the noise tool's range-based
  eligibility is deliberately preserved and covered by new unit tests.)
- Tie randomisation now uses the tight luminance threshold for **all** scalar modes, not just plain
  Brightness — Top % and Diff placements no longer scramble between blocks up to 24 luminance apart.

## Block list & screen polish

- Lists cap at 10½ rows — the half-cut row plus a slim scrollbar make it obvious there's more to
  scroll.
- A yellow key under every list explains the active buttons, switching to Pick-mode help when
  relevant.
- Pick mode rows are click-to-adjust: left-click +1, right-click −1 (0 removes the number).
- New shortcuts on every list: **double-click** un-excludes / toggles ✓, **middle-click** un-ticks /
  toggles exclusion.
