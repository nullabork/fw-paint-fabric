# FW Paint v1.5.0

The curve release: the easing curve moves into its own thin column on the Gradient and Noise tabs,
drawn as its actual shape — with a live distribution strip under it that you can **drag to reshape
the gradient yourself**. And the noise tool gets a curve at all, for the first time.

Targets (unchanged): Minecraft **26.2** · Fabric Loader **0.19.3+** · Fabric API for 26.2 (Fabric
jar only) · Java **25**.

## The curve strip

- A new thin middle column on the Gradient and Noise tabs (the outer columns just shift apart).
  At its top the **curve button**, now drawn as the curve's own shape instead of a word — Linear,
  Ease In, Ease Out, Ease In/Out, Step.
- Below it, the **distribution strip**: every gradient step stacked top (start) to bottom (end),
  drawn with its block iso-tiled, each band's height the share of the fill that step covers under
  the current curve. Black lines separate the bands.
- **Drag the handles** beside the strip (they alternate sides) to move a boundary: the band below
  grows, the one above shrinks, and the curve flips to **C — Custom**. Real placement uses exactly
  the shares you drew: gradient fills, noise fills, and both previews all run through the same
  boundaries, and Step-length wobble wobbles around them.
- Cycling the curve button snaps the strip back to that curve's automatic shape. Custom shapes are
  kept separately per tool; if the step count changes they fall back to even steps.

## Noise curve

- The noise tool now has its own **Curve** (it previously had none): it shapes how the noise value
  maps onto your block order, so an eased or custom curve makes some steps cover more of the
  pattern than others. Applied identically in placement and the cube preview.

## Small stuff

- Pick mode's key under the block list now actually explains itself: numbers are the placement
  order, click selects and +1s, right-click −1s, 0 = unused.
- The in-game Help covers the curve button, the strip, and the noise curve.
